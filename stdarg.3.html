Synopsis

#include <stdarg.h>

void va_start( va_list ap, rest);

type va_arg( va_list ap, type);

void va_copy( va_list dest, va_list src);

void va_end( va_list ap);

Description

Functions may be defined as accepting a variable number of arguments. The macros defined in <stdarg.h> permit stepping through the arguments passed to the called function.

An object of type va_list must be declared. This object is used by va_start(), va_arg(), va_copy() and va_end().

The va_list object is initialised by first calling the va_start() macro. The rest argument is the last fixed parameter of the function's prototype (that is, the parameter immediately preceding the ... varadic placeholder). Varadic functions must always be defined as requiring at least one fixed parameter. The rest parameter should not be declared as a register variable, a function, or an array type, because its address is used to determine the address of the parameter following it.

The va_arg() macro returns the value of the parameter pointed to by the va_list ap and advances the pointer to the next parameter. The type specified in the call to va_arg() is the expected type of the parameter passed by the caller. It shall be specified in such a way that the corresponding pointer type can be obtained by appending *.

If va_arg() is invoked more times than there are varadic parameters, or if the type specified does is not compatible with the type of parameter passed by the caller, the results are undefined.

The va_copy() macro copies the argument pointer src into dest. Applications must use va_copy() rather than directly assigning src to dest if they wish to store the current state of an argument pointer.

When a va_list object is no longer required, it must be freed with va_end().

Standards

<stdarg.h> conforms to the Single UNIX Specification, Version 3 (SUSv3).
